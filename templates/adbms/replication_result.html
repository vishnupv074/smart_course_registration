How It Works

This demo simulates a real-world scenario where data is written to a Primary database and asynchronously replicated to a Read Replica.

  1. Write to Primary: We updated the capacity of a Section on the Primary node.
  2. Read from Replica: Immediately after the write, we attempted to read the same record from the Replica node.
  3. Consistency Check: We compared the values. If they differ, it demonstrates Replication Lagâ€”the time it takes for changes to propagate.
Technical Details:
Primary Config: wal_level=replica
Replica Config: hot_standby=on
LSN (Log Sequence Number): A pointer to a location in the Write-Ahead Log. Comparing LSNs allows us to measure exactly how far behind the replica is.
